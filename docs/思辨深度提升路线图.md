## 🔴 Tier 1 — 重构产品定位（做一两个就能质变）

### 1. 成长时间线可视化

**核心问题**：用户四年里思维模式发生了怎样的变化？

**为什么现在没人能做**：只有你手里有四年对话数据。

**具体形态**：
- 按学期/学年展示关键词频率热力图（如"大二上学期最常提'焦虑'和'简历'，下学期'作品集'取代了'焦虑'"）
- 五维属性随时间的变化曲线
- 方向探索路径图（技术→设计→产品 的转向轨迹）
- AI 生成的年度成长总结："这一年你从纠结'哪个方向好'变成了思考'我适合什么'"

**技术可行性**：中等。依赖 localStorage 中积累的历史数据，需要数据聚合和可视化引擎。

---

### 2. 毕业后模式（企鹅生命周期扩展）

**核心问题**：四年结束后呢？企鹅就死了吗？

**具体形态**：
- 毕业后企鹅跟随用户进入「职场模式」
- 新增职场阶段外观（西装企鹅、创业企鹅、自由职业企鹅等）
- 聊天话题从「选课/社团/实习」切换为「跳槽/晋升/行业选择」
- 回顾大学四年的成长数据，生成「你的大学成长报告」

**为什么重要**：这是产品定位从「大学生工具」升级为「终身成长伙伴」的关键一步。

**技术可行性**：较高。核心引擎可复用，主要新增阶段外观和 System Prompt。

---

### 3. 决策回溯系统

**核心问题**：用户三年前选了 A 方向，今天回头看意味着什么？

**具体形态**：
- 记录用户每个重要决策（选专业、选实习方向、选 offer）的时间点和当时的理由
- 在后续某个节点触发回溯：「还记得三年前你为什么选这个方向吗？当时你说'因为喜欢创造东西'——现在你还在创造吗？」
- 决策 - 结果 关联分析，用你自己的历史数据验证你的判断力

**技术可行性**：中等。需要决策标注机制（手动或 AI 自动识别），以及结构化存储。

---

## 🟡 Tier 2 — 从「聊」到「做」

### 4. 行动转化引擎

**核心问题**：聊完"我想学 Python"然后呢？

**具体形态**：
- 当用户表达学习意图时，企鹅自动生成结构化计划（21 天/7 天/3 天）
- 每日检查进度，未完成时主动戳用户
- 完成后触发成就和解锁新配饰
- 支持「弹性计划」——根据实际执行情况动态调整

**技术可行性**：较高。核心是计划模板 + localStorage 状态跟踪 + 定时提醒（Notification API）。

---

### 5. 模拟面试官模式

**核心问题**：企鹅能不能不只是聊天伙伴，还是面试教练？

**具体形态**：
- 企鹅切换为「面试官」人格（严肃专业语气）
- 根据用户简历和目标岗位自动出题
- 用户语音/文字回答，企鹅从三个维度评价：内容结构、逻辑链条、表达清晰度
- 给出「如果是我，我会这样说」的参考答案

**技术可行性**：中等。需要新增 System Prompt 角色 + 评分逻辑。

---

### 6. 跨用户匿名对标

**核心问题**："和我同样方向的大三学生，平均水平是什么样的？"

**具体形态**（可选云同步）：
- 匿名聚合统计数据：同年级同方向的平均属性值、常见困惑、热门技能
- 「在你这个阶段，78% 的人最担心的是面试，但你已经在做作品集了——你领先了」
- 前提：需要匿名数据上传和隐私保护机制

**技术可行性**：低（需要后端）。可先做本地版的「理想路径对照」。

---

## 🟢 Tier 3 — 技术深度

### 7. 离线优先 + 本地模型

**核心问题**：校园网差/隐私敏感场景，能不能不依赖云端 API？

**具体形态**：
- 集成 WebLLM（如 WebGPU 驱动的本地小模型）
- 离线时自动降级为本地模型（响应质量降低但保持可用）
- 隐私敏感话题（如心理咨询倾向）默认走本地模型

**技术可行性**：低（当前 WebLLM 体积大、加载慢）。可关注 Gemini Nano 等浏览器内置方案。

---

### 8. 多模态输入

**核心问题**：用户在课堂上拍的笔记照片，能不能直接变成知识库？

**具体形态**：
- 拍照上传课堂笔记 → AI 提取关键点 → 自动归类到知识库
- 拍一张招聘海报 → AI 提取 JD 关键信息 → 关联到用户的技能差距分析
- 拍照识别教材封面 → 自动添加阅读计划

**技术可行性**：较高（依赖视觉模型 API）。

---

### 9. 自主 Agent 模式

**核心问题**：能不能不需要用户主动提问，企鹅自己知道什么时候该说什么？

**具体形态**：
- 企鹅定期（每周/每两周）主动发起话题："你上周说想学 Figma，这周有进展吗？我看到 Figma 出了新教程"
- 检测用户长期沉默后主动关心（不骚扰式）
- 在用户生日/学期开始/考试周/毕业季自动触发特定对话

**技术可行性**：中等。核心是定时器 + 状态检测 + 话题库。

---

### 10. API 化 / 可嵌入

**核心问题**：其他教育产品能不能嵌入这只企鹅？

**具体形态**：
- 把企鹅封装为可嵌入的 Web Component / iframe
- 提供 API 让第三方传入用户数据（年级、方向、目标），企鹅自适应
- 输出标准化成长报告（JSON/PDF）

**技术可行性**：较高。核心是模块化重构 + 接口标准化。

## 🎯 关键提醒

**下一次开发时，请优先阅读本文档的「P0 本周」部分。**

产品当前最核心的问题：**四年里的所有对话数据是一次性的——聊完就过去了，没有产生任何累积性洞察。** 成长时间线可视化是解决这个问题的第一步，也是最关键的一步。
