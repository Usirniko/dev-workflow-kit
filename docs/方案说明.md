# 🐧 AI 企鹅养成 · 方案说明

> 大学生思辨成长伙伴 —— 从产品构想到工程落地的完整记录

---

## 一、问题诊断

当前高校就业指导面临三重困境：**信息单向灌输缺乏互动**、**学生缺乏持续成长追踪**、**企业雇主品牌难以穿透校园场景**。学生在四年大学生涯中，从大一的好奇迷茫到大四的求职焦虑，缺少一个「始终在线」的陪伴者来引导思辨、沉淀成长轨迹。

具体痛点包括：新生入学后缺少行业认知启蒙；大二大三学生方向探索缺乏结构化引导；求职季学生面临 Offer 决策焦虑但无人对练；而企业侧则面临「如何提前 4 年建立情感链接」的长期雇主品牌命题。

---

## 二、方案设计

### 2.1 核心定位

打造一个「**企鹅养成 + AI 思辨对话**」的 Web 应用，以虚拟企鹅伙伴为载体，陪伴大学生四年成长。企鹅根据用户对话内容逐步成长进化，每一次深度交流都转化为企鹅的经验值——「你成长，鹅也成长」。

### 2.2 用户路径

**首次访问** → 领蛋页（Canvas 动画孵化）→ 年级选择 → 给企鹅命名 → 可选填写校招简历模板 → 进入主界面。

**日常使用** → 首页查看企鹅状态 → 进入聊天覆盖层与 AI 企鹅对话 → 聊天内容自动触发属性成长/活动解锁/装扮收集 → 日记/简历/知识库等辅助功能。

### 2.3 核心功能矩阵

| 系统 | 能力 |
|------|------|
| **聊天引擎** | DeepSeek API 调用 + 51 条离线问答库兜底 + 三级匹配策略 |
| **思辨引导** | 四个年级梯度化 System Prompt（好奇追问 → 认知偏差 → 第一性原理 → 决策矩阵） |
| **成长体系** | 五维属性（知识/创造/社交/行动/心态）+ 10 级等级 + 四阶进化 |
| **活动成就** | 20 个年级活动 + 5 项每日任务 + 10 个徽章 + 防刷机制 |
| **深度记忆** | 13 类话题识别 + 重要性评分 + 上下文注入 + 回访建议 |
| **主动关怀** | 沉默检测 + 50+ 关怀消息 + 企鹅气泡 UI + 不打扰策略 |
| **成长仪表盘** | 四 Tab 弹窗：时间线/技能树/职业雷达图/月度报告 |
| **日记 & 简历** | 日历视图日记 + 校招简历模板编辑 + 知识库（PDF/DOCX 解析） |

### 2.4 技术架构

**纯前端架构**：HTML5 + CSS3 + Vanilla JS（零框架依赖），Canvas 2D 渲染企鹅，localStorage 数据持久化，腾讯云 CloudBase 静态托管部署。无后端服务，API 调用直连 DeepSeek。

---

## 三、AI 工具选型理由

| 维度 | 选型 | 理由 |
|------|------|------|
| **大模型** | DeepSeek（`deepseek-chat`） | 中文能力强、性价比高、兼容 OpenAI 格式、支持自定义 Base URL |
| **兜底策略** | 离线问答库（51 条） | 无网络/无 API Key 时仍可对话，三级匹配保证可用性 |
| **文件解析** | PDF.js + Mammoth.js | 知识库上传解析，纯前端无需服务端 |
| **部署平台** | 腾讯云 CloudBase | 静态托管零运维、CDN 加速、自定义域名 |
| **开发工具** | CodeBuddy（AI 辅助编程） | 全流程 AI 辅助：需求分析 → 代码生成 → Bug 修复 → 部署 |

**选型核心原则**：最大限度降低用户使用门槛（无需注册/安装，浏览器即开即用）和部署运维成本（无服务器）。

## 四、关键配置

### 4.1 AI 模型参数

```
默认模型: deepseek-chat
默认端点: https://api.deepseek.com/v1/chat/completions
Temperature: 0.8
Max Tokens: 512
Timeout: 15000ms
Max Retries: 2（指数退避）
```

用户可在设置弹窗中自定义 API Key / Base URL / Model，全面兼容 OpenAI 格式接口。

### 4.2 数据持久化

5 个 localStorage 键：`penguin_data`（企鹅数据）、`chat_history`（聊天记录）、`user_config`（用户配置）、`app_state`（应用状态）、`resume_data`（简历数据），带安全读写封装和异常恢复。

### 4.3 安全规则

storage.js 提供 JSON 解析异常捕获、QuotaExceededError 自动清理、数据版本校验。API Key 使用 password 类型输入框，支持显示/隐藏切换。

## 六、效果评估

### 6.1 功能完整性

- ✅ 四阶段企鹅进化（蛋 → 幼崽 → 青少年 → 成年）
- ✅ 四个年级差异化思辨引导（14 条梯度规则）
- ✅ 五维属性 + 10 级等级 + 经验值体系
- ✅ 51 条离线问答库（三级匹配，无 API 也能聊）
- ✅ 消息质量六因子权重评估（0.3x~3.5x）
- ✅ 13 类话题深度记忆 + 主动回访建议
- ✅ 20 个年级活动 + 5 项每日任务 + 10 个成就徽章
- ✅ 21 个 JS 模块（~450KB 代码量）

### 6.2 工程质量

- 零 Lint 错误（19 个 JS 文件 + 1 个 CSS 文件）
- 全项目无 `TODO/FIXME/HACK` 遗留标记
- 异常处理覆盖：API 重试（5 类状态码分类）、存储溢出自动清理、Canvas 兜底、动画卡死保护
- 数据迁移兼容：日记 v1→v3 平滑迁移，存量数据零丢失

### 6.3 用户体验

- 首次访问引导路径 < 3 分钟完成
- 无 API 时可离线聊天，不受网络限制
- 移动端完全适配（viewport-fit + 触摸优化）
- 暗色模式支持

### 6.4 待优化方向

- `app.js` 单文件 ~185KB，建议按职责拆分模块
- 离线问答库可持续扩充至 100+ 条
- 引入自动化测试框架（当前为手工浏览器测试）
