# 项目开发流程总结

> 适用于纯前端 / 小工具 / 单文件应用类项目的通用开发方法论。

---

## 阶段一：需求文档化

### 1.1 产品需求文档（PRD）
写清以下内容：
- **产品定位**：一句话说清楚这是什么
- **目标用户**：谁在用、什么场景下用
- **核心功能模块**：按场景/页面拆解，每个模块列出关键交互
- **非功能性要求**：性能、平台、离线/在线等约束

### 1.2 数据结构设计
- 列出所有核心数据实体
- 每个实体定义字段名、类型、含义
- 明确实体之间的关系（一对一、一对多）
- 示例数据要写在文档里，方便后续直接拷贝到代码中

### 1.3 算法/规则说明
- 核心计算逻辑单独成文
- 用伪代码或自然语言描述计算步骤
- 标注边界情况和特殊处理

### 1.4 视觉设计规范
- 配色方案（主色、辅色、背景色、文本色）
- 字体与字号层级
- 组件样式约定（按钮、面板、弹窗的圆角/阴影/间距）

---

## 阶段二：技术选型与架构

### 2.1 技术选型
根据需求选择合适的技术栈：
- **纯前端单文件**：HTML + Canvas + 原生 JS（适合轻量级、离线场景）
- **框架应用**：React / Vue + UI 库（适合复杂交互）
- **全栈应用**：前后端分离 + 数据库（适合需要持久化、多用户场景）

### 2.2 架构设计
- 画出模块关系图
- 定义状态管理方案（全局 state 结构）
- 定义场景/路由切换逻辑
- 确定持久化方案（localStorage / IndexedDB / 后端 API）

---

## 阶段三：编码实现

### 3.1 实现顺序（推荐）
1. **骨架先行**：先搭 HTML/CSS 框架 + Canvas 画布初始化
2. **状态管理**：定义全局 state 对象和场景切换函数
3. **数据注入**：将文档中的核心数据转为 JS 常量
4. **逐场景实现**：启动页 → 主界面 → 核心玩法 → 结算页面
5. **弹窗/辅助模块**：设置、档案、提示等叠加层
6. **持久化**：存档/读档逻辑
7. **音效/动画**：最后抛光

### 3.2 编码原则
- 每个函数职责单一
- 弹窗类组件独立渲染，不与具体场景耦合
- UI 交互通过统一的注册机制，避免散落的点击事件
- 数据与逻辑分离：常量数据放配置区，逻辑放函数区
- **防御性编程**：关键流程全程 `try-catch` 包裹，防止单点故障导致页面卡死

---

## 阶段四：测试与修复

### 4.1 功能测试
按场景逐一跑通完整流程：
- 正常路径（完整走通全部页面）
- 异常路径（失败、中途退出、存档恢复）
- 边界情况（空数据、极限数值）

### 4.2 代码审查清单
- 检查 **JS 语法错误**
- 检查 **中文引号/特殊字符**（容易在字符串拼接时出问题）
- 搜索 **console.log / debugger** 残留
- 检查 **重复 ID / 重复调用**
- 检查 **废弃代码**（之前修改留下的冗余逻辑）
- 检查 **页面生命周期陷阱**：跳转/刷新时自动保存可能回写已清除的数据

### 4.3 常见陷阱

| 问题 | 表现 | 预防 |
|------|------|------|
| 弹窗与场景耦合 | 弹窗关闭后跳回错误页面 | 弹窗独立渲染，不随场景切换 |
| ID 重复 | 后注册的按钮不响应 | 每个交互元素用唯一 ID |
| 状态切换遗漏 | 切换场景后上一场景的 UI 残留 | 切换时清理中间状态 |
| 自动保存回写 | 重置数据后又出现旧数据 | 重置前设标记位，保存函数检测到标记后跳过 |
| CDN 缓存 | 部署后页面不更新 | 文件名带内容哈希，每次构建生成新文件名 |

## 阶段五：上线发布

### 5.1 项目整理
1. 补充 `.gitignore`（排除本地缓存、编辑器配置）
2. 写 `README.md`（项目介绍 + 快速开始 + 文档导航）
3. 确保所有文档与代码同步

### 5.2 Git 与 GitHub
```bash
git init                      # 初始化仓库
git add -A                    # 暂存所有文件
git commit -m "提交信息"        # 首次提交
git branch -M main            # 统一分支名
git remote add origin <URL>   # 关联远程仓库
git push -u origin main       # 推送到 GitHub
```

### 5.3 发布后的可选设置
- 添加仓库描述和 Topics（方便搜索）
- 如果是可部署的 Web 应用：启用 GitHub Pages 或云静态托管

---

## 流程速查

```
需求文档 → 数据设计 → 算法/视觉规范
    ↓
技术选型 + 架构设计
    ↓
骨架开发 → 状态管理 → 数据注入
    ↓
逐场景实现 → 弹窗/辅助模块 → 持久化
    ↓
全流程测试 → 代码审查 → 修复问题
    ↓
README + .gitignore → Git 提交 → 推送上线
```

---

## 适用场景

此流程适用于：
- 单文件/轻量级 Web 应用
- 小游戏/互动工具
- 原型验证项目
- 个人作品/演示 Demo

不适用于（需要额外步骤）：
- 多人员协作项目 → 需要分支策略 + Code Review 流程
- 后端服务项目 → 需要 API 设计 + 数据库迁移 + 部署运维
- 移动原生应用 → 需要平台适配 + 应用商店审核
